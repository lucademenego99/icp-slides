Interactive Code Playgrounds

A platform-agnostic technology to rethink
the programming slides user experience

Lorenzo Angeli and Luca De Menego
University of Trento
Berlin, June 1st 2022, OpenU WP3 Peer Learning Meeting

The typical programming lab slide
  • Code embedded in slides as images/plain text
  • Hard (or impossible) to copy/paste
  • As teacher, changes are hard to make
  • Not accessible for e.g., screen readers
Most importantly:
checking code results isn't easy!
Interactive Code Playgrounds
  • Run code directly on slide
  • Easy to modify/fix mistakes
  • No need to install anything

Other features in Interactive Code Playgrounds

Restricted Editing

Light Theme and Different Layouts

Other languages

Typescript (and Javascript)

Java

SQL

Development

Activity technical goal: make an MVP/proof of concept

Current solution is quite "tape-and-glue"

RevealJS for the slides

CodeMirror for the code editors

Pyodide for Python, Teavm for Java, SQL.js for SQLite

redbean for deployment

Pedagogical implications

Reduced learning curve for beginners (especially non-specialists)

Enable scaffolded learning

Smooth transition from traditional slides

Policy implications

Platform independence

Easy integration (not "yet another tool")

Promotes innovative learning (learning-by-tinkering)

More accessible (esp. for screen reader users)

Can work offline (doesn't exclude students with low internet access)
