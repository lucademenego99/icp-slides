Recursion

  • Should be familiar from imperative languages
  • In functional programming much more important; used as the main mechanism for iteration
  • Recursive function:
    • Basis: Compute teh result directly for sufficiently small argument
    • Inductive step: call the function recursively, with smaller arguments
  • Always make sure that there is a way for the recursion to terminate

Reversing a list

  • Base case: empty list to empty list
  • Induction: reverse the tail of the list (recursively) and then append the tail

Nonlinear recursion

  • A function can call itself recursively multiple times
  • Example: Number of combinations of m things out of n
    • Written n m
    • Can be shown to be equal to n ! n - m ! m ! , which would lead to a normal recursive definition
    • We shall use instead the following recursive definition

Combinations

  • Base case. if m = 0 the number of ways to pick 0 out of n is 0 .
    Similarly, if m = n , there is exactly one way to pick n out of n : n 0 = n m = 1
  • Induction. If 0 < m < n to select m out of n we can
    • Reject the first thing and select m out of the remaining n - 1
    • Pick the first thing, and pick m - 1 out of the remaining n - 1
    • Formally n m = n - 1 m + n - 1 m - 1
  • We use this to write a ML program

Mutual recursion

  • Two functions that call one another recursively
  • A function that takes a list L and produces a list with the first, third, fifth etc. elements of L
  • Two functions:
    • take(L) takes the first element of L and then alternates
    • skip(L) skips the first element and then calls take

Type inference

  • How ML deduces types
  • Types of operands and results of arithmetic expressions must agree
  • a + b ⋅ 2.0
    • The right operand is a real
    • Therefore, a + b must be a real
    • Therefore, so are a and b
  • In a comparison a < = 10 , both arguments have the same type, so a is an integer
  • In a conditional, the types of the then, the else and the expression itself must all be the same
  • If an expression used as an argument of a function is of a known type, the parameter must be of that type
  • If the expression defining the result of a function is of a known type, the function returns that type
  • If there is no way to determine the types of the arguments of an overloaded function (such as +), the type is the default (usually integer)

Example

  • Result of if clause is an integer
  • Therefore comb(n-1,m)+comb(n-1,m-1) is an integer
  • Therefore, so are comb(n-1,m) and comb(n-1, m-1)
  • Therefore comb returns an integer
  • The expressions n-1 and m-1 must be integers, so comb maps pairs of integers to an integer
