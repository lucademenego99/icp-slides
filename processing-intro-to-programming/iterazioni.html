Sintassi del ciclo for

// Sintassi del ciclo for
for (init; test; update) {
  statements
}

Il ciclo for è tipicamente utilizzato quando il numero di iterazioni è conosciuto a priori

test è l'espressione razionale, valutata true o false

Nelle parentesi graffe {} inseriamo il codice che va ripetuto

Il counter è tipicamente creato in init

  • Esso è una variabile che conta il numero di iterazioni
  • Esso viene tipicamente aggiornato in update
  • La stessa variabile è tipicamente utilizzata in test

Qualche consiglio

  • Si, i cicli possono essere infiniti (tipicamente un errore LOGICO del programmatore)
  • Ma in certi casi può essere utile avere un ciclo infinito
    • Quando può essere utile un ciclo infinito? Proponi un esempio...
  • E' possibile dichiarare il counter prima del ciclo
  • println() può essere utile se il ciclo non si comporta come previsto (vedi slide su Debugger)
  • Usa la variabile counter dentro il for! Per esempio, per disegnare qualcosa che dipende dal valore del counter

Differenza tra draw() e for

  • Non confondere un ciclo con un draw():
    • draw() può essere usato per dare l'impressione di movimento
    • for disegnerà tutto il prima possibile

Compara i due snippet. Quale credi sia la differenza?

void setup() {
  size(300, 300);
  for (int i = 1; i < 50; i++) {
    line(i*10,i*10,i*10+10,i*10);
  }
}
int i;
void setup() {
  size(300, 300);
}
void draw() {
  line(i*10,i*10,i*10+10,i*10);
  i++;
}

Sintassi del while

// Sintassi del while
while(test) {
  statements
}

while è tipicamente utilizzato quando il numero di iterazioni non è conosciuto a priori

Nelle parentesi graffe {} inseriamo il codice che va ripetuto

  • se viene valutata false, esci immediatamente dal ciclo e continua con il resto del programma (dopo })

// Sintassi del do while

while vs do...while

while

  • L'espressione è testata prima di un'iterazione
  • Il numero minimo di iterazioni è 0
  • Esci se l'espressione di test è false

do...while

  • L'espressione è testata dopo un'iterazione
  • Il numero minimo di iterazioni è 1
  • Esci se l'espressione di test è false

Cicli annidati

E' possibile inserire un ciclo all'interno di un altro

  • Il ciclo interno esegue un ciclo completo per ogni singola iterazione del ciclo esterno
  • quindi il ciclo interno è più "veloce"

Quindi, per ogni iterazione del ciclo esterno vengono eseguite tutte le iterazioni del ciclo interno

Tipicamente, utilizziamo due variabili counter

Confronta i due snippet

int a = 0;
for (int i= 0; i < 100; i=i+10){
  for (int j= 0; j < 100; j=j+10){
    a++;
  }
}
int a = 0;
for (int i= 0; i < 100; i=i+10){
  a++;
}
for (int j= 0; j < 100; j=j+10){
  a++;
}

  • Un for serve per controllare le coordinate x (cioè le colonne del disegno)
  • Un secondo for viene usato per controllare le coordinate y (cioè le righe del disegno)
