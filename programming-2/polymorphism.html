Polymorphism

  • Polymorphism (in general): the ability to assume different forms
  • Polymorphism (in programming languages): the ability of a syntactic element to refer to elements of different types
  • Liskov's substitution principle: if X is a subtype of T, then variables of type T in a program can be substituted for variables of type X without altering any desirable properties of the program

Polymorphism in Java

  • A variable of reference type T may refer to any object whose type is T or its subtype.
  • Similarly, a formal parameter of reference type T may refer to actual parameters whose type is T or one of its subtypes.

What is the purpose of this?

  • Inheritance and polymorphism are fundamental in object-orientation
  • Together, they make it possible to:
    • keep the definition of a class "open" to subsequent extensions (specialisations), without having to modify the code that uses it;
    • choose the most appropriate "view" to manage an object (general vs. specific) each time.
  • Strong positive impact on readability and maintainability of the code!

Dynamic binding: example

Static type and dynamic type

  • In the presence of polymorphism, a distinction is made between the static type (declared at compilation time) and the dynamic type (assumed at runtime) of a formal variable or parameter.
    • The substitution rule forces the dynamic type to be a subtype of the static type.
    • In Java, in an invocation x.f(x1, ..., xn), the implementation chosen for the method f depends also on the dynamic type of x and not only on its static type

Rule for dynamic binding

  • Let's assume to have:
    C o = ...;
o.m(...);
    the method chosen depends on the dynamic type of o, and is decided (at runtime) using the following logic:
    • We search inside the class C (static type of o) for the method m with the signature that is closest to the invocation.
      • The signatures to consider are those fixed at compile time, i.e. looking at the static type of the current parameters
    • We look at the dynamic type D of o; if it is a subtype of C, we have to check if it redefines (overrides) m. If yes, we use D's implementaion, otherwise C's.

The static type determines which methods can be invoked; the dynamic type determines which (re)definitions to perform.

class A { 
  private int val; 
  public A(int v) { val = v; } 
  public int value() { return val; } 
  public int sum(A o) { return value() + o.value(); } 
}

class B extends A { 
  B(int v) { super(v); } 
  public int sum(A o) { 
    return value() + o.value() + 2;
  } 
  public int sum(B o) {
    return value() + o.value() + 1;
  }
}
